# 踩坑分享（1 分半，PPT 第 11-12 页）

> 用时：约 90 秒 | 三个坑，每个 30 秒

---

做项目不可能一帆风顺，分享三个我们踩过的坑。

**第一个坑：Python 3.13 兼容性。**

我们一开始用 newspaper3k 做正文提取，结果一跑就报 `ModuleNotFoundError: sgmllib`。查了才知道 Python 3.13 把这个二十年的标准库移除了，而 newspaper3k 和它的依赖 feedparser 都还在用。

我们的解决是把整条抓取链换了——正文提取从 newspaper3k 换成 trafilatura，RSS 解析从 feedparser 换成 lxml 直接解析 XML。trafilatura 是个更现代的库，反而更好用。教训是：用新版本 Python 要格外注意底层依赖的兼容性。

**第二个坑：Ollama Python SDK 的幽灵 502。**

模型都拉好了，`ollama list` 正常，`curl` 调 API 也正常——就是 Python 的 `ollama` 库死活返回 502。翻 GitHub Issues 查了一晚上，发现是 SDK 的 HTTP 客户端在 Windows 上有问题。

解决方案很简单——不要用 SDK，直接用 `requests.post` 调 Ollama 的 HTTP API，一行代码就解决了。经验是：SDK 不是银弹，有时候裸 HTTP 调用更可靠。

**第三个坑：数据库"失踪"。**

页面一直显示"今日暂无数据"，明明是刚抓的文章。查了半天发现 SQLite 数据库文件有两个——一个在项目根目录，一个在 news_summary 目录。Streamlit 和命令行脚本从不同位置启动，`DB_PATH` 是相对路径，各自指向不同的文件。

改了一行代码——`DB_PATH` 改成基于 `config.py` 文件位置的绝对路径。以后不管从哪启动，数据库永远在同一个地方。

---

三个坑的共同特点：都不是算法问题，是工程问题。调试花的时间比写算法都多——但这恰恰是真实开发的常态。
