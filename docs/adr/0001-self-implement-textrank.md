# ADR 0001: 自己实现 TextRank 而非使用 sumy 库

## 状态

已采纳

## 背景

题目要求实现抽取式摘要方案（TextRank）。Python 生态有现成库 `sumy` 可直接调用。需要决定自己实现还是调库。

## 决策

自己实现 TextRank 算法（约 50-80 行），不依赖 sumy。

## 权衡

### 调库 (sumy)
- 优点：一行代码，零 bug 风险
- 缺点：答辩时只能讲"我调了个库"，无法展示对算法的理解；技术深度维度（25 分）可能失分

### 自己实现
- 优点：答辩可讲解 TF-IDF 向量化 → 余弦相似度 → PageRank 迭代全过程；体现课程知识（词向量、图算法）
- 缺点：多 50-80 行代码需要写和测试；PageRank 收敛需要调参

## 后果

- TextRank 实现需包含：中文分词（jieba）、句子 TF-IDF 向量化、余弦相似度矩阵构建、PageRank 迭代至收敛
- 需要和 sumy 做一次正确性交叉验证（用同一篇文章，两边输出应高度重叠），确保实现正确
